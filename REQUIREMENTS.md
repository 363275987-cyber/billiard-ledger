## 系统定位

台球教学的财务记账系统。核心场景：记录线下私域收入 + 电商（抖音/快手/视频号）销量，管理资金流转。

**核心原则：** 电商的钱只有提现到银行卡才算收入。

```
订单成交 → 待结算（店铺钱包） → 已提现（到银行卡） → 算收入
```

## 第1页：Dashboard（首页）

### 顶部卡片（3个）
1. **今日销售额** — 线下 + 电商全部订单的 payment_amount 之和（只是看今天卖了多少，不代表收入确认）
2. **今日订单数** — 线下 + 电商订单数
3. **本月利润** — 线下收入（含电商提现到账的金额）- 线下支出

### 下方两栏
- 左边：最近10笔订单（线下+电商混合，电商标平台标签）
- 右边：账户余额总览（个人+企业合计，不含电商）

### 数据来源
- 今日销售额/订单数：orders 表全量（platform_type IS NULL 或 NOT NULL 都算）
- 本月利润：线下收入 + 转账记录中电商→现金的金额 - 支出
- 电商提现后通过转账记录进入现金账户，自动算入利润

## 第2页：订单管理

### Tab切换
- 全部订单（默认）
- 私域订单（platform_type IS NULL）
- 电商订单（platform_type IS NOT NULL）

### 列表列
- 订单号、**店铺账户**、来源/平台标签、客户、商品名称、金额、状态、时间
- 电商订单标记平台标签（抖音/快手/视频号）
- 私域订单标记所属账户

### 操作
- 新建私域订单（手动）
- 导入电商Excel（按钮在页面顶部）
- 查看/编辑订单
- 退款跟着订单走（电商退款随售后Excel导入）

### 电商导入
- 支持：抖音/快手/视频号 销售订单 + 售后订单
- 未知SKU自动创建产品（用Excel里的名称+价格）
- 退款归属原订单成交日（order_time）
- 导入时自动 increment_balance 到对应店铺账户

## 第3页：支出管理

### 保持现状
- 手动登记支出（房租、工资、材料、物流、广告等）
- 支出分类管理
- 按分类/日期筛选
- 列表：日期、分类、金额、备注、账户

## 第4页：产品库

### 定位
不是"严格管理的主数据"，而是"自动积累的参考库"。
核心价值：订单列表显示产品名称、核对销量。

### 功能
- 产品列表：名称、SKU编码、分类、品牌、成本价、零售价
- 搜索（按名称或SKU）
- 按分类/品牌筛选
- 新建/编辑产品
- 从飞书表格同步（批量）

### 自动创建
- 电商导入时，遇到未知SKU自动创建新产品
- 用Excel里的商品名称+价格
- 不需要数据清理（去重、标准化）

## 第5页：电商销量

### 数据来源
- 通过电商店铺账户关联：一个账户 = 一个店铺
- 订单的 account_id 决定归属哪个店铺
- 销售额按 order_time（成交日）统计
- 退款归属原订单成交日

### 顶部汇总
- 待结算资金分店铺显示（每个店铺一个数字）

### 筛选
- Tab：全部 / 按平台（抖音、快手、视频号）
- 按店铺筛选（下拉或标签）
- 日期快捷选择（今日/7天/30天/本月）+ 自定义范围

### 图表
- 日趋势折线图（按平台分色，可叠加对比）
- 平台分布饼图

### 店铺明细
- 店铺/日期/平台/订单数/销售额/退款/净营业额
- 不需要订单明细

## 第7页：转账提现

### 保持现状
- 从电商店铺转到现金账户时，自动弹出费用明细面板
- 费用：技术费 + 支付费 + 提现费 + 其他费用
- 支持内扣（从本笔扣）和外扣（余额另扣）
- 费率：抖音 5%+0.6%+0.6%、快手 5%+0.6%+0.3%、视频号 2%+0.6%+0.1%

## 数据流转关系

```
产品库 ← SKU匹配 → 订单（私域+电商）
    ↓                     ↓
订单 → 金额 → 账户余额（私域→现金账户，电商→店铺账户）
    ↓
电商订单 → 电商销量页面（按店铺账户关联）
私域订单 → Dashboard收入统计
    ↓
转账提现 → 电商店铺余额 → 现金账户余额（这时才算收入）
    ↓
支出管理 → 记录开销 → 扣减现金账户余额
    ↓
Dashboard本月利润 = 私域收入 + 电商提现到账 - 支出
```
