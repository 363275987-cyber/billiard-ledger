# 台球账目系统 — 产品需求文档

> 版本：v2.1
> 日期：2026-04-01
> 说明：本文档描述功能和业务逻辑，不约束具体实现方式和UI版面

---

## 一、这是什么

一个给台球教学工作室用的记账和运营管理系统。主要解决以下几件事：

1. **私域收入**：学员通过微信/支付宝/银行卡付款的订单，手动记录
2. **电商收入**：抖音、快手、视频号三个平台的台球用品销售，从平台导出Excel批量导入
3. **支出管理**：日常经营开销的记录和分类
4. **库存和采购**：台球用品的库存管理和进货
5. **客户管理**：学员和客户的联系信息
6. **团队管理**：教练排课、课时记录、提成核算
7. **资产管理**：台球桌等固定资产

**最核心的业务规则：电商订单成交后钱在平台账户里，只有提现到银行卡才算真正的收入。** 系统要把"待结算的电商余额"和"已经到手的现金"严格分开。

## 二、系统模块和页面

### 核心模块（7个页面，第一优先级）

#### 1. 首页概览

一眼看到今天的经营情况和本月盈亏。

需要知道的信息：
- 今天总共卖了多少（私域+电商都算）
- 今天成交了多少笔
- 本月赚了多少（私域收入 + 电商已提现金额 - 本月支出 = 本月利润）
- 最近几笔订单是什么（方便快速回顾）
- 各个现金账户还有多少钱

注意：电商待结算的余额不算在"赚了多少"里面，因为它还没到手。

#### 2. 订单管理

所有订单的统一入口，支持手动录入（私域）和Excel批量导入（电商）。

**私域订单和电商订单是两种完全不同的东西，需要分开看：**

私域订单需要记录的信息：
- 客服号：是哪个微信在跟客户聊，方便追溯
- 收款账户：钱到了哪个账户（微信/支付宝/银行卡）
- 客户姓名和电话
- 卖了什么产品
- 多少钱
- 订单类型：是客户自己来的（自然成交）、销售引导来的、还是平分单（多个销售分业绩）
- 状态：待确认/已确认

电商订单需要记录的信息：
- 哪个平台的哪个店铺
- 卖了什么（商品名 + SKU编码）
- 多少钱
- 平台标签（抖音/快手/视频号）
- 订单状态（平台给的状态，如待发货/已完成/已退款等）

列表显示方式根据当前选的Tab不同而不同。用户可以在"全部"、"私域"、"电商"之间切换。

**电商Excel导入是核心功能，逻辑比较复杂：**

平台每个月会导出销售报表和售后报表，我们用Excel导入的方式把数据灌进来。

三个平台的Excel格式各不相同（抖音、快手、视频号），但都能从飞书表格上获取。需要做的事情：
1. 识别是哪个平台（看Sheet名）
2. 逐行解析，提取订单号、时间、店铺名、SKU、金额、状态
3. 如果这个订单号已经导入过了，跳过（去重）
4. 根据店铺名查找或自动创建对应的店铺账户
5. 根据SKU编码查找产品库，找到就关联上
6. 如果SKU编码在产品库里找不到，自动创建一个新产品（用Excel里的商品名和金额）
7. 把订单写进数据库，同时给对应店铺账户增加余额
8. 如果是售后报表，只导入"退款成功"的记录，匹配到原订单后扣减店铺余额

退款归属规则：某一天的销售额 = 当天成交的所有订单金额 - 这些订单的所有退款（不管退款发生在哪一天）。比如3月15号卖了一单500块，3月20号退了，那3月15号的净营业额就是0，而不是3月20号。

#### 3. 支出管理

记录日常开销。

支出分类：工资、房租、设备、营销/广告、税费、日常、物流、平台费、退款、其他

每笔支出要记录：花了多少钱、什么分类、付给了谁、从哪个账户扣的钱、备注

支持按日期和分类筛选。待确认的支出可以编辑和删除。

#### 4. 产品库

一个产品参考库，核心用途是订单列表里能显示产品名称，以及核对各SKU的销量和库存。

**产品库不是严格管理的主数据，而是"用着用着自动积累起来的"。** 比如导入电商订单时遇到一个从未见过的新SKU，就自动创建一个产品记录。不需要专门去做数据清洗、去重、标准化。

产品分两层：
- 产品（SPU）：一个产品有一个名称、分类、品牌
- SKU：一个产品下面可能有多个规格，每个规格有一个唯一的SKU编码

支持手动创建产品、编辑、搜索（按名称或SKU）、按分类筛选。

#### 5. 电商销量

电商业务的独立看板，专门看线上卖了多少。

核心功能：
- 按平台筛选（全部/抖音/快手/视频号）
- 按店铺筛选（下拉选择某个店铺）
- 按时间筛选（今日/近7天/近30天/本月/自定义范围）
- 汇总卡片：总销售额、总退款、净营业额
- 待结算资金：每个店铺各有多少钱还没提现，底部显示合计
- 日趋势图：按日期看的销售额/退款/净营业额折线图
- 店铺明细表：每个店铺每天的数据

不需要看到具体的每一笔订单（那是订单管理页的事），只需要按店铺按天的汇总数据。

**注意**：这里看到的销售额是"平台上的成交额"，不是到手收入。到手收入要看提现记录。

#### 6. 账户管理

管理所有资金账户。

账户分三类：
- 个人账户：微信、支付宝、个人银行卡等
- 企业账户：公司银行卡等
- 电商账户：抖音店铺、快手店铺、视频号店铺

电商账户的余额显示为"待结算资金"，**不计入现金总余额**。只有个人和企业账户的余额才算现金。

顶部有三个统计卡片，分别显示个人账户总余额、企业账户总余额、电商待结算资金总额。点击卡片可以筛选对应分类。

#### 7. 转账提现

管理资金在不同账户之间的转移。

两种场景：
1. **普通转账**：现金账户之间的转移，比如从微信转到银行卡
2. **电商提现**：从电商店铺提现到银行卡，这时会自动计算平台手续费

平台手续费规则：
- 抖音：技术服务费5% + 支付通道费0.6% + 提现手续费0.6%
- 快手：技术服务费5% + 支付通道费0.6% + 提现手续费0.3%
- 视频号：技术服务费2% + 支付通道费0.6% + 提现手续费0.1%

当从电商店铺提现时，需要显示费用明细（每项扣多少），让用户知道实际到手多少。扣费方式支持"内扣"（从本笔金额中扣）和"外扣"（从账户余额另扣）。

### 运营模块（6个页面，第二优先级）

#### 8. 客户管理

管理所有客户和学员的联系信息。

每个客户需要记录：
- 姓名、电话、微信
- 来源渠道（怎么知道我们的：抖音、快手、小红书、朋友介绍、其他）
- 状态（活跃/沉默/流失）
- 备注信息

支持按姓名/电话搜索，按来源渠道和状态筛选。可以看到客户的订单历史（点击客户名跳转到订单页并过滤该客户）。

#### 9. 退款管理

独立的退款记录页面，方便集中查看所有退款情况。

退款信息：
- 关联的原订单（订单号、客户名、商品名）
- 退款金额
- 退款原因
- 退款状态（待处理/已退款）
- 退款时间

支持按时间范围、平台、退款状态筛选。可以查看退款对应的原订单详情。

电商退款在导入售后Excel时自动创建，私域退款需要手动录入。

#### 10. 客服号管理

管理工作室用于客服工作的微信号。

每个客服号需要记录：
- 微信号/微信号ID
- 绑定的客服人员
- 当前状态（在用/停用）

主要用于私域订单中标记"这个订单是哪个微信在跟进"，方便追溯客户是从哪个渠道来的。

#### 11. 库存管理

管理台球用品的库存。包含四个子功能：

**仓库管理：**
- 记录仓库信息（名称、地址）
- 管理不同仓库的库存分布

**库存总览：**
- 按产品/SKU查看当前库存数量
- 库存预警（低于某个数量时提醒补货）
- 按分类/品牌筛选

**库存流水：**
- 每次库存变动都有记录（入库、出库、盘点调整）
- 记录变动原因和操作人
- 按时间、SKU筛选

**采购订货：**
- 创建采购订单（向供应商订货）
- 订单状态流转：待确认 → 已确认 → 已发货 → 部分到货 → 已到货
- 到货后自动入库

#### 12. 教练与学员管理（Coaching）

管理教练团队和学员的课时记录。

**教练管理：**
- 教练基本信息（姓名、联系方式、擅长领域）
- 排课/空闲时间
- 课时统计（本月上了多少课）

**学员管理：**
- 学员基本信息
- 课程等级（初级/中级/高级）
- 购买的课时包（剩余课时）
- 上课记录

**课时记录：**
- 哪个教练给哪个学员上了课
- 上课时间、时长
- 课时扣减（从学员的课时包里扣）

#### 13. 资产设备管理

管理工作室的固定资产，主要是台球桌。

每个资产需要记录：
- 资产名称、编号
- 购入日期、购入价格
- 当前状态（在用/维修/报废）
- 存放位置

资产分组功能：比如把4张球台分成一组，方便批量管理。

折旧自动计算（可选）。

### 系统管理（2个页面，仅管理员可见）

#### 14. 用户管理

管理系统账号和权限。

- 邀请新用户（通过邮箱/手机号）
- 设置用户角色（管理员/财务/经理/销售/客服/仓库/教练）
- 不同角色看到不同的页面和操作权限

#### 15. 系统设置

全局配置。

- 支出分类管理（可以自定义分类，不限于预设的那些）
- 平台费率配置（如果将来平台费率变了，可以在这里改）
- 其他业务参数

## 三、关键业务规则汇总

1. **电商的钱不算收入**：只有提现到银行卡才计收入。电商余额是"待结算资金"
2. **退款归属原订单日**：查某天的净营业额时，要把那些订单的所有退款都扣掉，不管退款发生在哪天
3. **待发货订单也算余额**：电商订单导入时，只要金额大于0就给店铺加余额（包括待发货的）
4. **部分退款也要算收入**：部分退款的订单仍然计为"已完成"状态，金额按实际到手的来
5. **产品库自动积累**：遇到新SKU就自动创建，不需要人工维护
6. **订单类型（私域专属）**：自然成交、销售引导、平分单——用来区分业绩归属
7. **库存扣减和回补**：电商订单成交时自动扣减对应SKU的库存，退款成功时自动回补
8. **课时自动扣减**：教练给学员上完课后，自动从学员的课时包里扣减对应课时

## 四、数据流转关系

```
电商Excel导入 → 订单表 + 店铺余额 + 产品库 + 退款表
手动录入订单 → 订单表 + 现金账户余额 + 产品库

订单表 ──────────→ 首页（今日数据）
                  → 电商销量（按平台/店铺/日期聚合）
                  → 退款管理（退款记录）
                  → 客户管理（客户订单历史）

产品库 ──────────→ 订单管理（产品选择）
                  → 库存管理（库存显示）

支出管理 ────────→ 支出表 + 对应账户余额

账户管理 ←──────── 订单导入（余额变动）
         ←──────── 转账提现（余额变动）
         ←──────── 支出管理（余额扣减）

转账提现 ────────→ 账户余额（电商→现金，扣手续费）

库存管理 ←──────── 采购到货（自动入库）
         ←──────── 电商订单（自动出库）
         ←──────── 电商退款（自动回补）

教练学员 ────────→ 课时记录 → 课时包扣减
```

## 五、角色和权限

不同角色看到的页面不同：

- **管理员**：全部页面
- **财务**：核心7页 + 系统管理（不能改用户）
- **经理**：核心7页（不能看系统管理）
- **销售**：订单管理 + 客户管理 + 客服号管理
- **客服**：订单管理（只读） + 客户管理
- **仓库**：库存管理（仓库/库存/流水/订货）
- **教练**：教练学员管理（自己的学员和课时）

## 六、技术要求

- 前端：Vue 3，用 Tailwind CSS 做样式
- 后端：Supabase（PostgreSQL数据库）
- 样式风格参考 Keep 运动App：绿色主色调、圆角卡片、大字数字显示
- 已有的数据库和Supabase配置不变，重建时复用现有数据库结构
